# Trinity Guard — A 股多智能体量化交易系统

三系统（政策 / 量化 / 审计）15 角色协同博弈，专为中国 A 股短线设计。

## 架构

```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  System A    │  │  System B    │  │  System C    │
│  政策高度     │  │  量化执行     │  │  对抗审计     │
│  5 角色      │  │  5 角色      │  │  5 角色      │
└──────┬───────┘  └──────┬───────┘  └──────┬───────┘
       │ 密封结论         │ 密封结论         │ 密封结论
       └────────────┬─────┴─────────────────┘
                    ▼
            ┌──────────────┐
            │  MetaJudge   │
            │  中央仲裁官    │
            └──────┬───────┘
                   ▼
            ┌──────────────┐
            │  硬风控层      │  ← 宪法级，AI 不可改写
            └──────┬───────┘
                   ▼
            ┌──────────────┐
            │  执行层       │  ← LocalPaper / 同花顺 / 东方财富
            └──────────────┘
```

## 三大核心指数

| 指数 | 作用 | 输出 |
|------|------|------|
| **MSI** 市场强度 | 进场漏斗 + 持仓风格切换 | 强攻 / 观望 / 防守 |
| **CGI** 资金博弈 | 博弈态判断 | 一致进攻 / 独角龙 / 补涨末端 / 拥挤陷阱 |
| **LRI** 流动性风险 | 硬否决 | 通过 / 否决 |

## 三时段状态机

| 时段 | 任务 |
|------|------|
| 9:25-9:35 | 紧急熔断检查（只产生熔断信号） |
| 10:10-10:20 | 主状态定性（决定今日进攻权限） |
| 13:30-14:00 | 尾盘再确认 |
| 收盘后 | 复盘 + T+1 结算 |

## 快速开始

### 1. 环境准备

```bash
cp .env.example .env
# 编辑 .env，填入 Tushare Token 和 One-API Key
```

### 2. 启动 One-API 网关

```bash
docker-compose up -d one-api
# 访问 http://localhost:3888 配置模型渠道
```

### 3. 安装依赖

```bash
pip install -r requirements.txt
```

### 4. 本地模拟运行

```bash
python -m src.main
```

### 5. 切换券商后端

编辑 `config/broker.yaml`：

```yaml
active_broker: ths    # local_paper / ths / dfcf
```

## 项目结构

```

│   ├── models.yaml          # LLM 模型路由
```

## 定时任务（cron）

```cron
# 盘前启动（9:15）
15 9 * * 1-5 cd /path/to/Trinity-Guard && python -m src.main >> data/logs/cron.log 2>&1

# 盘后复盘已集成在 main.py 流程中
```

## 核心原则

1. **硬风控高于 AI** — 宪法级规则不可被学习层改写
2. **执行与学习解耦** — 白天执行，晚上复盘，进化需验证
3. **认知异质性** — 三系统追求不同世界观，不是人数规模
4. **试探仓先行** — 首仓永远是计划仓位的 1/3
5. **学错误来源** — 复盘只学"为什么亏"，不学短期盈亏情绪
